# Discord Event Bot

Бот для объявлений об ивентах с подпиской на уведомления в ЛС.

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
```

В `.env` укажите `DISCORD_TOKEN`. Опционально `GUILD_ID` для быстрой синхронизации команд.

## Запуск

**Бот:**
```bash
python main.py
```

**Веб-панель** запускается вместе с ботом (`main.py`), если в `.env` задан `WEB_PASSWORD`.  
Отдельно только сайт: `python run_web.py`.

Откройте http://127.0.0.1:8080 — вход по `WEB_USERNAME` / `WEB_PASSWORD`.  
Отключить: `WEB_ENABLED=false`.

В панели: список ивентов, редактирование, подписчики ЛС, со-хостеры, организаторы, отмена/завершение. Изменения синхронизируются с объявлением в Discord (нужен `DISCORD_TOKEN`).

## Время и Discord-таймер

Время задаётся текстом по МСК:
- `ЧЧ:ММ` — сегодня
- `ДД.ММ.ГГГГ ЧЧ:ММ` — конкретная дата

В объявлениях отображается встроенный таймер Discord:
- `<t:...:F>` — полная дата
- `<t:...:R>` — обратный отсчёт («через 2 часа»), обновляется сам

Команды:
- `/ивент создать` — поле **время** в формате выше
- `/ивент перенос` — перенос с пингом и ЛС подписчикам
- Изменение времени — кнопки под объявлением (**Управление ивентом**)

## Роли и права

Назначение только через `.env` (перезапуск бота после изменения):

| Переменная | Кто |
|------------|-----|
| `ADMIN_ROLE_IDS`, `ADMIN_USER_IDS` | Админы ивентов (все ивенты) |
| `ALLOWED_ROLE_IDS`, `ALLOWED_USER_IDS` | Ивент-хостеры (свои ивенты + со-хостеры) |

Discord-администраторы сервера тоже считаются админами ивентов.

### Со-хостеры
На объявлении: **Управление ивентом** → **Со-хостеры**

## Команды

- `/ивент создать`, `/ивент список`, `/ивент отменить`, `/ивент перенос`, `/ивент завершить`
- `/ивент организаторы` — список хостеров и админов

## Запуск из Cursor (чтобы агент видел логи)

1. Откройте терминал в Cursor: `` Ctrl+` ``
2. Запустите бота:
   ```powershell
   cd "c:\Users\foldy\Desktop\bot test"
   py main.py
   ```
3. Напишите в чат: «проверь логи бота» — агент прочитает вывод терминала.

Чтобы терминал не блокировался, можно запустить в фоне или второй вкладкой терминала.
